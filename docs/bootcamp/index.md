# 부트캠프 백엔드 개발자 편 with 스프링 부트

## 1. 스프링, 스프링 부트와의 첫 만남

- 프레임워크와 라이브러리는 모두 다른 사람이 작성한 코드다.
- 프레임워크와 라이브러리의 가장 큰 차이점은 코드를 호출하는 지점(제어 흐름)이다.
- 프레임워크는 정해진 흐름 속에서 사용자의 코드를 호출한다. 라이브러리는 사용자가 직접 호출한다.
- 스프링부트(Spring Boot)는 스프링(Spring)을 설정 중심에서 사용 중심으로 변화한 도구다. 복잡한 설정을 자동화로 처리했다.
- 컴퓨터로 비유하면 스프링은 내부 구조, 동작 원리에 가깝고, 스프링 부트는 컴퓨터를 실제 사용하는 경험에 가깝다.
- 컴퓨터를 처음 사용했을 때 CPU, RAM, 회로부터 배웠다면 금방 흥미를 잃었을 것이다. 마찬가지로 스프링도 우선 스프링부트를 통해 사용 경험을 쌓고 자연스럽게 내부 원리(스프링)를 접하는 방법을 권장한다.

## 2. 스프링 부트와 웹

- 인터넷과 웹은 비슷하게 사용한다. 하지만 명백하게 다른 의미다.
- 인터넷은 전 세계 컴퓨터를 연결하는 거대한 네트워크 인프라다.
- 웹은 인터넷을 사용하여 정보를 공유한다.
- 웹은 인터넷 위에서 동작하는 수많은 서비스 중 하나다.
- 인터넷이 도로라면 웹은 인터넷 위에서 달리는 자동차와 같다.
- HTTP는 HTML 문서와 같은 리소스를 가져올 때 사용하는 프로토콜이다.
